# Contributing to CodeLens AI

Thank you for your interest in contributing to CodeLens AI! We welcome contributions from the community to help improve our platform.

## Code of Conduct

Please read and follow our [Code of Conduct](CODE_OF_CONDUCT.md) to ensure a welcoming environment for all contributors.

## How Can I Contribute?

### Reporting Bugs

Before submitting a bug report, please check if the issue has already been reported. If not, create a new issue with:

- A clear and descriptive title
- Steps to reproduce the issue
- Expected vs actual behavior
- Screenshots or code examples if applicable
- Environment details (OS, browser, etc.)

### Suggesting Enhancements

Feature requests are welcome! Please submit an issue with:

- A clear and concise description of the proposed feature
- Explanation of the problem it solves
- Examples of how it would work
- Any alternative solutions considered

### Code Contributions

#### Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/codelens-ai.git`
3. Create a new branch: `git checkout -b feature-name`
4. Make your changes
5. Commit your changes: `git commit -m "Add feature"`
6. Push to your fork: `git push origin feature-name`
7. Submit a pull request

#### Development Setup

See our [Development Guide](docs/development-guide.md) for detailed instructions on setting up your development environment.

#### Coding Standards

- Follow the existing code style
- Write clear, self-documenting code
- Include unit tests for new functionality
- Update documentation as needed
- Ensure all tests pass before submitting PR

#### Pull Request Process

1. Ensure your changes adhere to our coding standards
2. Update the README.md with details of changes if applicable
3. Increase the version numbers in any examples files and the README.md to the new version that this Pull Request would represent
4. Your PR will be reviewed by maintainers who may request changes
5. Once approved, your PR will be merged

## Styleguides

### Git Commit Messages

- Use the present tense ("Add feature" not "Added feature")
- Use the imperative mood ("Move cursor to..." not "Moves cursor to...")
- Limit the first line to 72 characters or less
- Reference issues and pull requests liberally after the first line

### JavaScript/React

- Use functional components with hooks
- Follow the Airbnb JavaScript style guide
- Use PropTypes for type checking
- Keep components small and focused

### Python/FastAPI

- Follow PEP 8 style guide
- Use type hints where possible
- Write docstrings for public methods
- Keep functions small and focused
